# 追加のコンテキスト機能

> 📖 **前提条件**: この付録を読む前に[第 02 章: コンテキストと会話](../02-context-conversations/README.md)を完了してください。

この付録では 2 つの追加コンテキスト機能をカバーします: 画像の操作と複数ディレクトリにわたる権限管理です。

---

## 画像の操作

`@` 構文を使って会話に画像を含めることができます。Copilot はスクリーンショット、モックアップ、図表、その他のビジュアルコンテンツを分析できます。

### 基本的な画像の参照

```bash
copilot

> @screenshot.png この UI で何が起きていますか？

# Copilot が画像を分析して応答する

> @mockup.png @current-design.png これら 2 つのデザインを比較してください

# 画像をドラッグ＆ドロップするか、クリップボードから貼り付けることもできる
```

### サポートされている画像フォーマット

| フォーマット | 最適な用途 |
|--------|----------|
| PNG | スクリーンショット、UI モックアップ、図表 |
| JPG/JPEG | 写真、複雑な画像 |
| GIF | シンプルな図表（最初のフレームのみ） |
| WebP | ウェブスクリーンショット |

### 実用的な画像の使用例

**1. UI のデバッグ**
```bash
> @bug-screenshot.png ボタンが正しく配置されていません。どの CSS が原因でしょうか？
```

**2. デザインの実装**
```bash
> @figma-export.png このデザインに合わせた HTML と Tailwind CSS を書いてください
```

**3. エラーの分析**
```bash
> @error-screenshot.png このエラーはどういう意味ですか？どうすれば修正できますか？
```

**4. アーキテクチャのレビュー**
```bash
> @whiteboard-diagram.png このアーキテクチャ図をドキュメントに入れられる Mermaid 図に変換してください
```

**5. ビフォー・アフターの比較**
```bash
> @before.png @after.png この UI の 2 つのバージョンで何が変わりましたか？
```

### 画像とコードの組み合わせ

画像はコードのコンテキストと組み合わせるとさらに強力になります:

```bash
copilot

> @screenshot-of-bug.png @src/components/Header.jsx
> スクリーンショットではヘッダーがおかしく見えます。コードのどこが原因ですか？
```

### 画像のヒント

- **スクリーンショットをトリミングする** — 関連する部分だけ表示する（コンテキストトークンを節約できる）
- **高コントラストを使う** — 分析してほしい UI 要素には高コントラストが効果的
- **必要に応じてアノテーションを付ける** — アップロード前に問題箇所を丸で囲んだりハイライトしたりする
- **1 つのコンセプトに 1 枚の画像** — 複数の画像も使えるが、焦点を絞ること

## 権限パターン

デフォルトでは、Copilot は現在のディレクトリのファイルにアクセスできます。他の場所のファイルにはアクセス権を付与する必要があります。

### ディレクトリの追加

```bash
# ディレクトリを許可リストに追加する
copilot --add-dir /path/to/other/project

# 複数のディレクトリを追加する
copilot --add-dir ~/workspace --add-dir /tmp
```

### すべてのパスを許可する

```bash
# パス制限を完全に無効にする（慎重に使用すること）
copilot --allow-all-paths
```

### セッション内での操作

```bash
copilot

> /add-dir /path/to/other/project
# これで別のディレクトリのファイルを参照できる

> /list-dirs
# 許可されているすべてのディレクトリを確認する

> /yolo
# /allow-all のクイックエイリアス — すべての権限プロンプトを自動承認する
```

### 自動化のために

```bash
# 非インタラクティブなスクリプトのためにすべての権限を許可する
copilot -p "Review @src/" --allow-all

# または覚えやすいエイリアスを使う
copilot -p "Review @src/" --yolo
```

### 複数ディレクトリアクセスが必要なとき

これらの権限が必要となる一般的なシナリオ:

1. **モノレポ作業** — パッケージ間でコードを比較する
2. **クロスプロジェクトのリファクタリング** — 共有ライブラリを更新する
3. **ドキュメントプロジェクト** — 複数のコードベースを参照する
4. **移行作業** — 古い実装と新しい実装を比較する
